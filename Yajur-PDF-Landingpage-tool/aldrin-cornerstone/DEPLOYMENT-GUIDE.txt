1. FILES INCLUDED

2. REQUIRED CHANGE BEFORE DEPLOYING
------------------------------------
In the index.html file, there are 3 references to "aldrin-cornerstone.vercel.app"
that MUST be updated to your actual domain where this page will be hosted.

HOW TO DO IT:
  - Open index.html in any text editor
  - Use Find & Replace (Ctrl+H or Cmd+H)
  - Find:    aldrin-cornerstone.vercel.app
  - Replace: YOUR-DOMAIN.com
  - Replace All (it will find 3 matches)

For example, if hosting at listings.aldrin.ca:
  - Find:    aldrin-cornerstone.vercel.app
  - Replace: listings.aldrin.ca

The 3 lines that will be updated:
  Line 8:   <base href="https://aldrin-cornerstone.vercel.app/">
  Line 252:  src="https://aldrin-cornerstone.vercel.app/logo-dark.png"
  Line 626:  src="https://aldrin-cornerstone.vercel.app/logo-dark.png"

IMPORTANT: If hosting under a subdirectory (e.g. aldrin.ca/cornerstone/),
the base href must include the subdirectory:
  <base href="https://aldrin.ca/cornerstone/">


3. SERVER DEPLOYMENT
--------------------
This is a simple static site. No build step, no npm install, no database
setup required. Just place the 2 files on your web server.

STEP 1: Upload both files to the same directory on your server
          /var/www/your-site/index.html
          /var/www/your-site/logo-dark.png

STEP 2: Make the change described in Section 2 above

STEP 3: Ensure your web server serves index.html as the default page

-- NGINX Configuration Example --

  server {
      listen 80;
      server_name your-domain.com;
      root /var/www/your-site;
      index index.html;

      location / {
          try_files $uri $uri/ /index.html;
      }
  }

-- APACHE Configuration --

  No special config needed. Apache serves index.html by default.
  Just place both files in your web root or desired directory.

-- IIS (Windows Server) --

  Place both files in the site's root directory.
  index.html is served as default document.


4. ALTERNATIVE: REVERSE PROXY SETUP
------------------------------------
If you prefer to proxy the existing Vercel deployment instead of
hosting the files locally, no file changes are needed.

NGINX reverse proxy example:

  location /cornerstone/ {
      proxy_pass https://aldrin-cornerstone.vercel.app/;
      proxy_set_header Host aldrin-cornerstone.vercel.app;
      proxy_set_header X-Real-IP $remote_addr;
      proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
      proxy_set_header X-Forwarded-Proto $scheme;
  }

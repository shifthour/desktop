

as you know this is EDC system for clinical trails , in form builder page , The goal is to design a Demographics eCRF form where a Study Designer can:

Choose which fields to include/exclude (e.g., Date of Birth, Sex, Race, Ethnicity, Height, Weight, Consent Date).

Define field types (text, integer, decimal, date, dropdown, radio button, checkbox, multi-select, file upload, calculated).

Configure validations (required, regex, ranges, cross-field rules).

Define conditional visibility rules (e.g., Pregnancy field only visible if Sex = Female).

Map each field to CDISC DM standard variables.

Output the form definition as structured JSON that can be rendered dynamically in the UI.

Generate:

A recommended set of fields for Demographics with suggested field types (dropdown vs. radio, etc.).

A sample JSON schema representing the form definition (fields, types, validations, mappings).

Example UI controls (e.g., show Sex as a radio button, Race as a multi-select dropdown, Date of Birth as a date picker).

Editable options for the Study Designer so they can add/remove fields easily.

Make sure the design is modular, reusable, and compliant with 21 CFR Part 11 (audit trails, required flags, versioning).

We already have a Form Builder, Studies tab, and Subjects tab. Now we want to add a "Queries" tab. 

The Queries tab should support both auto-generated queries (from validation rules) and manual queries (raised by Data Manager or CRA). 

Requirements for the Queries tab:
1. Display a table/grid view of queries with columns:
   - Query ID
   - Study
   - Subject ID
   - Visit
   - Form
   - Field
   - Value entered
   - Query text/message
   - Severity (Error/Warning/Note)
   - Status (Open, Answered, Closed)
   - Raised by (system/DM/CRA)
   - Assigned to (CRC/PI)

2. Query workflows:
   - Open → Assigned → Answered → Reviewed → Closed
   - CRC/PI should be able to respond to queries.
   - DM/CRA should be able to close queries after review.
   - Auto-queries are created when validation fails during data entry.

3. Actions:
   - Raise Manual Query (select form field, enter query text, assign to site user)
   - Respond to Query (CRC/PI enters response)
   - Close Query (DM/CRA after review)
   - Filter/Sort queries by study, subject, status, severity, date.

4. Compliance:
   - Every action (open/respond/close) must be logged in the audit trail with user, role, timestamp, and action.
   - Reason for query closure must be captured.

5. UI considerations:
   - Tab should show a list of all queries.
   - Clicking a query opens a detail panel with full context (subject, form, field, history, and responses).
   - Include search and filter options (status, severity, raised by).
   - Export option (CSV/XLSX/PDF) for query listings.

Please generate the full design/implementation for this Queries tab, including data model, UI components, and workflow logic. Assume it will connect with the existing Subjects/Forms data entry module.

As we are building Electronic Data Capture (EDC) system for clinical trials. The Form Builder and Studies Tab are already implemented. Now, design and generate the Subjects Tab module, which manages participants in a clinical study.

Requirements:

Subjects List View

Show all subjects for the selected study.

Columns: Subject ID, Screening Date, Enrollment Status (Screened / Enrolled / Completed / Withdrawn), Last Visit Date.

Actions: Add New Subject, Edit Subject, View Subject Timeline.

Add New Subject

Auto-generate Subject ID (e.g., SUBJ001, SUBJ002).

Capture basic details (e.g., Screening Date, Initials).

Assign the subject to the selected study.

Status defaults to Screened.

Subject Detail View

Show subject timeline with scheduled visits (Screening, Baseline, Follow-up, Close-out).

Each visit links to its assigned forms (from the Form Builder).

Visit status: Not Started / In Progress / Completed / Locked.

Data Entry Integration

CRC/PI can open a form for a given visit and enter data (Demographics, Vital Signs, etc.).

Show validation (e.g., error if Age > 120).

Allow Save as Draft and Final Submission.

Permissions

CRC: Add/edit subjects, enter data.

PI: Approve/lock forms.

DM/CRA: View subjects, review data, raise queries (but cannot edit data).

POC Demo Flow

User goes to Studies Tab → Select Study → Subjects Tab → Add Subject.

Subject timeline appears → Screening visit → Demographic form opens → Data entered → Validation triggered → Data saved.

Subject status updated automatically.

Output Format

Provide a React component with Tailwind CSS (or equivalent modern UI).

Use placeholders/dummy data for now, but keep the structure ready for integration.

Show clean navigation (tabs: Studies → Subjects → Subject Timeline → Forms).

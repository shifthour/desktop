We already have Form Builder, Studies tab, Subjects tab, and Queries tab. 
Now we want to add an "Audit Trail" tab to ensure compliance with 21 CFR Part 11 and GCP.

Requirements for the Audit Trail tab:

1. Display a log table/grid with the following columns:
   - Audit ID
   - Study
   - Subject ID (if applicable)
   - Visit (if applicable)
   - Form & Field (if applicable)
   - Action Type (Create, Update, Delete, Query Opened, Query Closed, Lock, Unlock, Promotion, etc.)
   - Old Value
   - New Value
   - User
   - Role
   - Timestamp (date & time)
   - Reason for Change (mandatory for edits)

2. Events to be captured:
   - Data entry (insert, update, delete).
   - Query lifecycle actions (open, respond, close).
   - Form version promotions (Draft → UAT → Production).
   - Subject lifecycle changes (screened, enrolled, withdrawn).
   - Database lock/unlock.
   - User role or permission changes.

3. Features:
   - Filters: by study, subject, form, date range, user, action type.
   - Search bar for quick lookup.
   - Export options: CSV, XLSX, PDF.
   - Ensure immutability: once written, audit logs cannot be modified or deleted.

4. UI:
   - Table view for listing logs.
   - Click on a log entry to open a detail panel showing full context (before/after values, reason for change, who performed the action).
   - Pagination for large datasets.

5. Compliance:
   - All changes must be linked to the specific user ID and role.
   - Timestamp should be precise (down to seconds).
   - Reason for change is mandatory for any data update.
   - Audit Trail tab must be accessible to Study Admin, Data Manager, CRA, and auditors (read-only).

Please generate the design and implementation for the Audit Trail tab, including:
- Suggested database schema/tables for storing audit logs.
- Frontend UI components (log table, filters, detail view).
- Backend logic for capturing audit events automatically.
- Export functionality for regulators.
